Reference

파일별 역할

각 파일의 책임과 레이어 분류입니다.

src/App.tsx
UI 선택 상태(selectedNoteId, isCreating) 관리. NotesProvider로 전체 트리를 감싸고 Layout에 슬롯 전달.
UI · Entry
src/context/NotesContext.tsx
서버 동기 상태(notes, loading, error) 관리. CRUD 메서드 제공. 낙관적 업데이트 패턴 적용.
State · Context
src/api/notes.ts
JSON Server(:3001) fetch 래퍼. 모든 HTTP 통신의 단일 진입점. 타임스탬프 클라이언트 주입.
API · Boundary
src/components/Layout.tsx
헤더 + 사이드바 + 메인 2단 레이아웃. 순수 구조 컴포넌트, 데이터 없음.
UI · Layout
src/components/NoteList.tsx
useNotes()로 목록 구독. loading / error / empty 세 가지 early return 처리.
UI · Consumer
src/components/NoteItem.tsx
단일 노트 표시. 상태 없는 순수 표시 컴포넌트. 클릭·삭제 이벤트를 props로 위임.
UI · Pure
src/components/NoteEditor.tsx
생성·편집 폼. 로컬 폼 상태(title, content, saving) 보유. addNote / editNote 호출.
UI · Consumer
src/types/note.ts
Note 인터페이스 단일 정의. id·title·content·createdAt·updatedAt. tags는 미구현.
Type · Shared
